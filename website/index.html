How It Works

Get started in just a few simple steps

1

Configure Telex Integration

First, go to the Integrations section in Telex and create a new integration. Provide the below URL and save to install the integration

https://telex-server-monitor.umoru.tech/integration-config
2

Run Setup Command in Telex

After installing the integration , activate the integration and run the setup command in the Telex channel you want to monitor.

/setup-monitoring
3

Copy & Run Installation Command

Telex will provide an installation command with your unique channel ID. Run this command on your server with sudo access.

curl -sSL https://raw.githubusercontent.com/telexintegrations/server-monitor-telex-integration/refs/heads/dev/install-telex-sdk.sh | bash -s -- --channel-id <your-channel-id>
4

Automatic Installation Process

The installation script will automatically:

  • Install Node.js and PM2 if they're not already on your server
  • Install the Telex Server Monitor package globally
  • Configure the monitoring agent with your channel ID
  • Set up PM2 to ensure the agent runs on system boot
  • Start the monitoring service immediately
5

Verify Installation

Check that the monitoring agent is running correctly on your server.

pm2 status telex-server-monitor
6

Start Receiving Alerts

That's it! You'll now receive real-time alerts and periodic reports in your Telex channel whenever your server needs attention.

Available Commands

/cpu Get current CPU metrics

Shows detailed CPU usage statistics for your server

/cpuLoadAvg Get CPU load averages

Displays CPU load averages over time

/perCoreUsage Get CPU usage metric per core

Displays CPU usage metric for each individual core

Note: These commands can be run at any time in your Telex channel to get immediate server metrics. The system also sends automatic alerts when critical thresholds are exceeded.

Managing Your Monitoring Service

After installation, you can manage your monitoring service using these PM2 commands:

pm2 status telex-server-monitor

Check if the monitoring service is running

pm2 logs telex-server-monitor

View real-time logs from the monitoring service

pm2 restart telex-server-monitor

Restart the monitoring service if needed

pm2 stop telex-server-monitor

Temporarily stop the monitoring service

Troubleshooting Tips

  • If the service isn't running, try restarting with pm2 restart telex-server-monitor
  • Check logs for any errors with pm2 logs telex-server-monitor
  • Logs are also stored in /var/log/telex-server-monitor/
  • If changes don't appear to be working, ensure PM2 startup is configured: pm2 save
