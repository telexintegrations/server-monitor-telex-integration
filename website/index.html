Getting Started is Easy!

Follow these simple steps to set up your server monitor.

1. Link Up with Telex

Head over to Telex's Integrations section, create a new one using the URL below, and hit save. Easy peasy!

https://system-integration.staging.telex.im/server-monitoring-integration/integration-config

2. Kick Things Off in Telex

Once installed and activated, jump into your chosen Telex channel and run this simple command to start the setup.

/setup-monitoring

3. Install on Your Server

Telex will give you a special command with your channel ID. Copy it, then run it on your server (you'll need sudo access).

curl -sSL https://raw.githubusercontent.com/telexintegrations/server-monitor-telex-integration/refs/heads/dev/install-telex-sdk.sh | bash -s -- --channel-id <your-channel-id>

4. Sit Back & Relax (Almost!)

The script works its magic, installing everything needed:

  • Installs Node.js and PM2 (if missing).
  • Grabs the Telex Server Monitor package.
  • Sets up the agent with your channel ID.
  • Ensures it runs automatically on boot using PM2.
  • Starts the monitor right away!

5. Quick Status Check

Just to be sure, run this command on your server to confirm the monitor is up and running smoothly.

pm2 status telex-server-monitor

6. You're All Set! 🎉

That's it! Sit back and enjoy the peace of mind. You'll now get real-time alerts and handy reports right in your Telex channel.

Keeping Your Monitor Healthy

Basic Service Controls (via Terminal)

Use these commands directly on your server to manage the monitoring service process.

Check Status

pm2 status telex-server-monitor

See if the monitor is running.

View Logs

pm2 logs telex-server-monitor

Check for errors or activity.

Restart

pm2 restart telex-server-monitor

Turn it off and on again.

Stop

pm2 stop telex-server-monitor

Temporarily pause monitoring.

Start

pm2 start telex-server-monitor

Resume monitoring if stopped.

Save Startup

pm2 save

Make sure PM2 starts on boot.

Alerts & Reports (via Telex AI Chat)

Control when and how you get notified directly by talking to the AI assistant in your Telex channel.

CPU Alert Thresholds

Default is 80%. Ask the AI: "Set CPU warning alert to 75%" or "Change critical CPU alert to 90%". Includes smart 5-min cooldowns.

Memory Alert Thresholds

Default is 85%. Ask the AI: "Set memory warning alert to 80%" or "Change critical memory alert to 95%". Includes smart 5-min cooldowns.

Scheduled Reports

Default is every minute. Ask the AI: "Send reports every 10 minutes" or "Disable scheduled reports". Uses cron syntax.

Advanced Controls (via Terminal)

Rarely needed, but these commands provide deeper control over the monitor installation.

telex-server-monitor help See all available monitor commands.
telex-server-monitor reset Reset all monitor settings to their defaults.
telex-server-monitor start Manually start the monitoring service (if not using PM2).
telex-server-monitor stop Manually stop the monitoring service (if not using PM2).

Quick Fixes

  • Service not running? Try restarting: pm2 restart telex-server-monitor
  • Seeing errors? Check the logs: pm2 logs telex-server-monitor
  • Logs also stored here: /var/log/telex-server-monitor/ and ~/.telex-monitor/logs
  • Not starting on boot? Ensure PM2 is saved: pm2 save
